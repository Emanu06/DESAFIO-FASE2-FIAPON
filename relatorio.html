SEC 01

Cenário de Pouso e Definição dos Módulos

A missão Aurora Siger consiste no estabelecimento da primeira colônia humana permanente em Marte. A região escolhida é a planície de Amazonis Planitia, selecionada por sua topografia plana, ausência de atividade geológica recente e proximidade com depósitos de gelo subsuperficial — essenciais para a extração de água e produção de oxigênio.

O MGPEB coordena a chegada sequencial de sete módulos de infraestrutura. Cada módulo possui atributos operacionais que determinam sua posição na fila, prioridade de atendimento e condições mínimas de segurança para autorização de pouso.

Princípio AND completo: qualquer valor False em qualquer uma das cinco variáveis propaga a negação até a saída, bloqueando o pouso. O sistema falha de forma segura (fail-safe) — padrão obrigatório em sistemas críticos embarcados.
SEC 04

Protótipo Python — MGPEB

O protótipo foi desenvolvido inteiramente em Python 3, sem bibliotecas externas além de collections (padrão da linguagem). Oito blocos funcionais cobrem todos os requisitos da atividade.

Bloco Central — Autorização de Pouso (Portas Lógicas)

mgpeb.py — autorizar_pouso()
def autorizar_pouso(modulo, atm_ok, area_ok, sensores_ok):
    """
    Implementa a função booleana de autorização de pouso.
    AUTORIZADO = (comb_ok AND massa_ok) AND (atm_ok AND area_ok) AND sensores_ok
    """
    combustivel_ok = modulo["combustivel"] >= 40   # Porta AND — variável 1
    massa_ok       = modulo["massa"]      <= 24.0  # Porta AND — variável 2
 
    seg_modulo  = combustivel_ok and massa_ok        # AND gate — bloco 1
    cond_ext    = atm_ok and area_ok                  # AND gate — bloco 2
    decisao     = seg_modulo and cond_ext and sensores_ok # AND gate — final
 
    return decisao, combustivel_ok, massa_ok

Bloco de Simulação — Processamento da Fila

mgpeb.py — simular_pouso()
while fila_pouso:
    modulo = fila_pouso.popleft()  # Remove o primeiro da fila (FIFO)
 
    autorizado, comb_ok, massa_ok = autorizar_pouso(
        modulo, CONDICOES_ATMOSFERICAS, AREA_POUSO_DISPONIVEL, SENSORES_OK

    if autorizado:

Algoritmos de Busca

BUSCA LINEAR — por tipo
O(n)
Percorre toda a estrutura comparando o campo "tipo". Retorna todos os módulos do tipo especificado. Simples e adequado para o tamanho reduzido do dataset.
BUSCA — menor combustível
O(n)
Percorre a lista mantendo referência ao menor valor encontrado. Identifica rapidamente módulos em situação crítica de abastecimento.
BUSCA BINÁRIA — por horário
O(log n)
Requer lista previamente ordenada por horário. Divide o espaço de busca pela metade a cada iteração. Resultado: MED-01 encontrado em horário 10h.
mgpeb.py — busca_binaria_por_horario()
def busca_binaria_por_horario(lista_ordenada, horario_alvo):
    esquerda, direita = 0, len(lista_ordenada) - 1
    while esquerda <= direita:
        meio = (esquerda + direita) // 2
        h = lista_ordenada[meio]["horario"]
        if   h == horario_alvo: return meio          # Encontrado
        elif h <  horario_alvo: esquerda = meio + 1  # Busca à direita
        else:                   direita  = meio - 1  # Busca à esquerda
    return -1                                         # Não encontrado

Algoritmos de Ordenação

BUBBLE SORT — Prioridade
O(n²)
Compara pares adjacentes e troca se necessário. Adequado para o dataset de 7 módulos. Ordena da maior para menor prioridade (1 = urgente).
SELECTION SORT — Combustível
O(n²)
Encontra o mínimo em cada passagem e posiciona no início. Útil para identificar módulos com risco de combustível, exibidos em ordem crescente.
INSERTION SORT — Horário
O(n²) / O(n)*
Eficiente para listas quase ordenadas — típico em janelas de pouso onde pequenas atualizações de horário ocorrem frequentemente. *O(n) no melhor caso.

SEC 05

Modelagem Matemática Aplicada ao Pouso

Para apoiar as decisões de engenharia do MGPEB, modelamos o fenômeno físico da descida livre do módulo na atmosfera marciana, desde a liberação orbital até a ativação dos retrofoguetes.

5.1 Função de Altitude em Função do Tempo

Com velocidade vertical inicial nula, a altura h(t) em função do tempo é dada pela equação cinemática de queda livre — uma função quadrática:

h(t) = h0 − ½ · g · t²

5.2 Gráfico de h(t)

5.3 Análise Qualitativa

A curva é uma parábola invertida — a velocidade de descida aumenta continuamente. Quanto maior h₀, mais tempo o módulo tem para desacelerar, o que é vantajoso para módulos mais pesados que precisam de maior impulso de frenagem.

Decisão de engenharia: quando h(t) ≤ 900 m (30% de h₀), o MGPEB emite o comando de ignição dos retrofoguetes. Esse limiar garante desaceleração suficiente para o módulo mais pesado (LOG-01, 22 t), que precisa atingir velocidade de toque segura (< 2 m/s). Módulos mais leves (MED-01, 7,8 t) acionam os retrofoguetes em altitude menor, economizando combustível para operações futuras.

O comportamento quadrático também demonstra que pequenas variações na altitude inicial (perturbações orbitais) têm impacto relevante no tempo disponível para a sequência de pouso — reforçando a necessidade do monitoramento contínuo dos sensores, variável já contemplada na equação booleana do sistema.

SEC 06

Contextualização Histórica da Computação

O MGPEB reflete uma longa trajetória evolutiva que tornou viáveis sistemas embarcados de missão crítica como os utilizados em sondas e rovers interplanetários.

1945 — 1948
Primeiros computadores de propósito geral
ENIAC (1945) e Manchester Mark 1 (1948) ocupavam salas inteiras, consumindo dezenas de quilowatts e sendo programados por fios e cartões perfurados. Representavam o ponto de partida de uma trajetória rumo à miniaturização radical.
1950s — 1960s
Transistores e circuitos integrados
A transição de válvulas para transistores e depois para circuitos integrados reduziu drasticamente tamanho, consumo e custo. Abriu o caminho para computadores que cabem em um foguete — ou dentro de um módulo espacial.
1969
Apollo Guidance Computer (AGC) — o precursor do MGPEB
Com apenas 4 KB de RAM e 72 KB de memória de núcleo magnético, o AGC guiou a missão Apollo 11 para a superfície lunar. Prova histórica de que restrições severas de hardware não impedem sistemas robustos — desde que algoritmos e estruturas de dados sejam escolhidos com critério.
1971
Intel 4004 — o microprocessador
O primeiro microprocessador comercial consolidou a era dos sistemas embarcados: computadores dedicados a uma função específica, integrados diretamente ao equipamento que controlam. Antecessor direto dos processadores de bordo de missões como Curiosity e Perseverance.
2012 — 2021
RAD750 — processador da era marciana
O processador RAD750, utilizado no rover Curiosity (2012) e Perseverance (2021), opera a 200 MHz — uma fração dos GHz dos chips comerciais — e é certificado para suportar doses de radiação 200.000× maiores que os limites humanos. Cada ciclo de processamento custa energia preciosa gerada por RTG.

Questões de Governança Específicas

Critérios de Escolha da Área de Pouso

A área de pouso deve ser selecionada por um processo deliberativo que combine análise técnica automatizada (sensores do MGPEB) com avaliação humana especializada. Os critérios incluem: impacto científico mínimo (distância de locais de potencial valor geológico ou astrobiológico), estabilidade geotécnica (ausência de deslizamentos recentes), visibilidade solar para os painéis fotovoltaicos e acessibilidade logística entre módulos pousados.

Gestão Sustentável de Recursos

A colônia opera em modo de economia circular marciana: CO₂ atmosférico → metano (Sabatier) → combustível de retrofoguetes. Eletrólise da água subterrânea → O₂ para respiração + H₂ para energia. Resíduos sólidos → filamento para impressão 3D estrutural. A geração elétrica combina painéis solares de alta eficiência com RTG de backup, garantindo autonomia durante as frequentes tempestades de poeira marcianas (que podem durar semanas e reduzir a irradiância solar em até 99%).

Mecanismos de Transparência e IA Explicável

O MGPEB não pode operar como uma "caixa-preta". Todas as regras de decisão são documentadas em linguagem acessível a qualquer membro da equipe — não apenas engenheiros. Cada autorização ou bloqueio de pouso gera um log imutável transmitido para a Terra e arquivado localmente em sistema redundante. Esse compromisso concretiza o princípio da IA Explicável (XAI) aplicado a sistemas críticos: quem é afetado por uma decisão automatizada tem o direito de entendê-la.

Estruturas de Dados: Detalhamento

Este anexo descreve em detalhes como cada estrutura linear foi utilizada no protótipo MGPEB, com exemplos concretos de operações sobre os módulos da missão.

A.1 Fila (Queue) — collections.deque

mgpeb.py — fila de pouso
from collections import deque
 
fila_pouso = deque()
 
# Inserção — O(1) — todos os módulos entram pelo fim
fila_pouso.append({"nome": "HAB-01", "prioridade": 1, "combustivel": 72})
fila_pouso.append({"nome": "ENER-01", "prioridade": 2, "combustivel": 55})
 
# Remoção — O(1) — retira sempre o primeiro (FIFO)
proximo = fila_pouso.popleft()
print(f"Processando: {proximo['nome']}")   # → HAB-01
print(f"Restam: {len(fila_pouso)} módulos")  # → 6
 
# Vantagem do deque vs list: list.pop(0) é O(n) — deque.popleft() é O(1)

A.2 Pilha (Stack) — list

mgpeb.py — pilha de alertas
pilha_alertas = []
 
# Empilhamento — O(1)
pilha_alertas.append({"nome": "LOG-01", "combustivel": 30})
pilha_alertas.append({"nome": "LOG-02", "combustivel": 20})
 
# Desempilhamento LIFO — O(1) — trata o mais recente primeiro
urgente = pilha_alertas.pop()

# Busca linear: menor combustível na lista de pousados
def busca_menor_combustivel(lista):
    menor = lista[0]
    for m in lista[1:]:
        if m["combustivel"] < menor["combustivel"]:
            menor = m
    return menor
 
# Bubble Sort: reordenar por prioridade
def bubble_sort_prioridade(lista):
    n = len(lista)
    for i in range(n):
        for j in range(0, n - i - 1):
            if lista[j]["prioridade"] > lista[j+1]["prioridade"]:
                lista[j], lista[j+1] = lista[j+1], lista[j]
    return lista
